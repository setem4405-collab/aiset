# AISET Program

This directory contains program-level documents defining the mission, scope, structure, governance, publication model, and long-term development direction of the AISET Program.

## Purpose

The AISET Program provides the institutional and engineering framework within which AISET publications, specifications, schemas, registries, governance records, and reference implementations are developed.

Program-level documents define:

- mission and purpose;
- architectural scope;
- program principles;
- publication authority;
- governance structure;
- roles and responsibilities;
- contribution processes;
- review and approval procedures;
- intellectual property and licensing direction;
- program evolution.

## Planned Program Documents

The first planned program publication is:

- `PROGRAM-0001` — AISET Program Charter.

Future program documents may define:

- governance procedures;
- contribution requirements;
- publication policies;
- review and approval models;
- terminology governance;
- registry authority;
- conformance policy;
- implementation policy;
- amendment procedures;
- dispute resolution;
- program succession and continuity.

## PROGRAM-0001

`PROGRAM-0001 — AISET Program Charter` is intended to establish the formal foundation of the AISET Program.

It should define, at minimum:

- the name and identity of the program;
- the mission of AISET;
- the engineering problem addressed by AISET;
- the scope and boundaries of the program;
- the relationship between publications, specifications, schemas, registries, and implementations;
- foundational principles;
- governance roles;
- publication authority;
- contribution and review processes;
- licensing principles;
- amendment and versioning procedures.

## Program Principles

The AISET Program is guided by the following principles:

1. Technology Independence;
2. Context First;
3. Shared Context;
4. Cognitive Compatibility;
5. Cognitive Interoperability;
6. Cognitive Portability;
7. Explicit Trust and Provenance;
8. Independent Measurement;
9. Governed Context Evolution;
10. Transparent Publication and Review.

## Roles

### Program Originator

The Program Originator defines the foundational mission, direction, and architectural intent of AISET.

### Author

An Author creates substantive program, architectural, technical, or governance content.

### Lead Editor

The Lead Editor maintains consistency of terminology, structure, publication quality, and program coherence.

### Reviewer

A Reviewer evaluates documents, specifications, implementations, or governance records against their stated purpose and requirements.

### Publisher

The Publisher authorizes and records official AISET releases.

Additional roles may be defined as the program develops.
