# PROGRAM-0001 Registry

This directory contains registry records associated with `PROGRAM-0001 — AISET Program Charter`.

Registry records preserve the registered identity, lifecycle status, canonical location, version relationships, review basis, supersession relationships, and publication state of PROGRAM-0001 documents.

---

## 1. Directory Purpose

This directory may contain:

- publication identifiers;
- version registrations;
- lifecycle status records;
- canonical artifact references;
- review and decision relationships;
- supersession relationships;
- checksums;
- release metadata;
- correction and amendment records.

A registry record documents the state of a registered object at the time of registration.

Registration alone does not establish final approval unless the applicable record explicitly states that approval has been granted through the required AISET Program process.

## 7. Integrity and Immutability

Once committed and published, a registry record must not be silently modified or replaced.

Material changes require one of:

- a new registry record;
- a correction record;
- an amendment;
- a superseding registry record.

Registered documents and their registry records must remain independently discoverable.

Repository history may provide supporting provenance but does not replace explicit registry records.

Candidate Publication requires cryptographic checksum evidence and registry binding under the Charter.

## 9. Status Interpretation

An active Review registry record means that the registered document is the current formal Review version.

It does not mean that the document is:

- finally approved;
- an Approved Publication;
- the governing Charter;
- independently validated;
- externally certified;
- an AISET Reference Publication.

Any lifecycle progression must be supported by the applicable review, integrity, registry, decision, and Publisher records.
